# Mutability?
- Strings are not mutable, need to re-assign: s = s + xyz or s += xyz
- Lists are mutable: list.append()

# Sorting a List
- sorted() sorts the item and returns a new one, does not change the original list
- sort() stores the new list into the original
- list.sort(reverse=True), words are now placed in descending order

# Making copies
## of stirngs
- If strings a = s, same IDs. If one change, only the modified one will change. Global (outside function) stays the same.
## of lists
- Stored under the same IDs. If one change, the other changes too. Global will have a problem.
- b = a[:] or b = a.copy() makes a copy that does not change globally

# Split
- .split() tokenizes lists based on spaces; converting data into words
- .split(' ') splits based on inside of the parenthesis
